Task Execution Flow

How CLAWG uses your Second Brain on every task

  1. 1. Detect intent

    The agent classifies the request (coding, research, ops, content, etc.).

  2. 2. Read shared context

    Loads master markdown files and agent profile (`identity`, `adjoint`, `soul`).

  3. 3. Select capabilities

    Searches vault skills/subagents and picks reusable playbooks before execution.

  4. 4. Execute with common tools

    Runs on the same global tools and API layer as all other agents.

  5. 5. Write back learnings

    Persists decisions, outputs, and learnings to vault markdown for future sessions.

Quickstart

Setup CLAWG + shared Obsidian vault in minutes

1. Install

curl -fsSL https://raw.githubusercontent.com/gquthier/CLAWG/main/scripts/install.sh | bash

2. Link your Second Brain

clawg second-brain link --path "/Users/gauthier/Documents/Second Brain OpenClaw - PROD"

3. Initialize structure

clawg second-brain init --agent-id founder

4. Verify resolved paths

clawg second-brain status

5. Launch with profile

clawg --agent-id founder

Where do I edit context files?

Inside your Obsidian vault. CLAWG reads markdown directly from the linked Second Brain path.

Do all agents share the same tools and skills?

Yes. The shared vault + global tools layer ensures every agent can discover the same capabilities.

Can I keep per-agent identity while sharing everything else?

Yes. Each agent has dedicated `agents/<id>/identity.md`, `adjoint.md`, and `soul.md` while using the same master context and capability pool.
