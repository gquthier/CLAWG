Why SMART-CLAWG

Your agents deserve a brain.

Single Source of Truth

All agents read from one Obsidian vault. No duplicated prompts. No stale context files scattered across machines.

Live Obsidian Editing

Edit any .md file in Obsidian. Changes are picked up at the next agent turn. No rebuild, no restart.

Multi-Agent, Same Brain

Founder, ops, dev, content — every agent loads its own identity but shares skills, tools, and learnings.

Obsidian Sync = Global Context

Enable Obsidian Sync and your agents on any machine share the same vault. Edit on laptop, run on server.

Shared Tools & Skills

Skills gallery, subagent catalog, API registry, learning journal — all visible to all agents, always.

Open Source, Self-Hosted

MIT licensed. Runs on your machine. No cloud dependency. No vendor lock-in. Fork it, extend it, own it.

Execution Flow

What happens when you ask.

01

Detect intent

Agent classifies the request: coding, research, ops, content.

02

Load shared context

Master files + agent profile loaded from the shared vault.

03

Match capabilities

Skills, subagents, and learnings queried before execution.

04

Execute

Runs with the global tools layer. Same API for every agent.

05

Write back

Learnings and decisions persisted to the vault for future sessions.

Quickstart

Five commands. That's it.

1

Install

pip install clawg
2

Link your vault

clawg second-brain link --path "~/My Second Brain"
3

Initialize templates

clawg second-brain init --agent-id founder
4

Check status

clawg second-brain status
5

Launch

clawg --agent-id founder
